stratum-sqlite

Load and query a read-only SQLite database on any static website — plain HTML, Quarto, Jekyll, Hugo, and more. No server. No backend. The file is cached in the browser after the first download.

Quick start — plain HTML

<!-- 1. Copy sql.js files into your project (see README) -->
<script src="libs/sqljs/sql-wasm.js"></script>
<script src="libs/sqljs/stratum-sqlite.umd.js"></script>

<script type="module">
  const db = await StratumSQLite.open("data/mydb.sqlite", {
    sqlJsPath: "libs/sqljs/",   // where sql-wasm.wasm lives
    cacheKey:  "mydb@v1",       // bump version to force re-download
  });

  const rows = db.query("SELECT * FROM countries");
  console.log(rows);

  const tables = db.tables();   // list all tables
</script>

Quick start — Quarto / ObservableJS

<!-- In _sqljs-init.html (injected via _quarto.yml) -->
<script>
  window._dbPath    = "data/mydb.sqlite";
  window._sqljsBase = "libs/sqljs/";
  window._sqlJsReady = new Promise((res, rej) => {
    const s = document.createElement("script");
    s.src = window._sqljsBase + "sql-wasm.js";
    s.onload = () =>
      initSqlJs({ locateFile: () => window._sqljsBase + "sql-wasm.wasm" })
        .then(res).catch(rej);
    s.onerror = () => rej(new Error("sql.js load failed"));
    document.head.appendChild(s);
  });
</script>

// In your .qmd file (OJS cell):
db = {
  const SqlJs = await window._sqlJsReady;
  const r     = await fetch(window._dbPath);
  const raw   = new SqlJs.Database(new Uint8Array(await r.arrayBuffer()));
  return { query: sql => {
    const res = raw.exec(sql);
    if (!res.length) return [];
    const { columns, values } = res[0];
    return values.map(row =>
      Object.fromEntries(columns.map((c, i) => [c, row[i]])));
  }};
}

Hosting your own database

Your SQLite file can live anywhere that serves files with Access-Control-Allow-Origin: *:

  • GitHub Releases — free, unlimited size, CORS-enabled
  • GitHub Pages — same origin, no CORS header needed
  • Zenodo / Figshare / OSF — public data repositories
  • Cloudflare R2 / AWS S3 — set a bucket CORS policy

For files that change regularly, change the cacheKey version whenever you publish a new release. The old cached file is evicted automatically.
